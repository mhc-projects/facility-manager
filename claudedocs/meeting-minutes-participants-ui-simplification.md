# 회의록 참석자 UI 간소화

## 🎯 개선 목표

참석자 섹션 UI를 단일 행으로 간소화하여:
- ✅ 이름 입력 필드가 명확히 돋보이도록 개선
- ✅ 혼동을 주는 추가 입력 필드 제거
- ✅ 직책 정보는 배지 형태로 간결하게 표시

## 📊 개선 전후 비교

### 개선 전 (2행 구조)

```
┌────────────────────────────────────────────────────┐
│ [참석자 이름 검색 또는 입력] [참석☑] [삭제]       │
│ [직책 입력 (선택사항)      ] [외부]               │ ← 혼동 요소
└────────────────────────────────────────────────────┘
```

**문제점**:
- ❌ 2줄 구조로 복잡함
- ❌ "직책 입력 (선택사항)" 필드가 이름 입력과 혼동
- ❌ 외부 참석자의 불필요한 입력 필드
- ❌ 이름 입력 필드가 덜 돋보임

### 개선 후 (단일 행 구조)

```
┌──────────────────────────────────────────────────────────────┐
│ [참석자 이름 검색 또는 입력...] [내부·팀장] [참석☑] [삭제]  │
└──────────────────────────────────────────────────────────────┘
```

**장점**:
- ✅ 1줄로 간결하고 명확
- ✅ 이름 입력 필드가 시각적으로 돋보임
- ✅ 직책은 배지로 표시 (입력 혼동 없음)
- ✅ 깔끔한 시각적 계층 구조

## 🎨 구현 세부사항

### 레이아웃 구조

```tsx
<div className="flex items-center gap-3 p-3 bg-gray-50 rounded-lg">
  {/* 1. 이름 자동완성 입력 (flex-1: 최대 공간 차지) */}
  <div className="flex-1">
    <AutocompleteSelectInput
      placeholder="참석자 이름 검색 또는 입력..."
      // ...
    />
  </div>

  {/* 2. 직책/구분 배지 (flex-shrink-0: 고정 크기) */}
  {participant.name && (
    <div className="flex items-center gap-2 flex-shrink-0">
      {participant.is_internal ? (
        <>
          <span className="badge-internal">내부</span>
          {participant.role && (
            <span className="text-sm text-gray-600">{participant.role}</span>
          )}
        </>
      ) : (
        <span className="badge-external">외부</span>
      )}
    </div>
  )}

  {/* 3. 참석 체크박스 (flex-shrink-0: 고정 크기) */}
  <label className="flex items-center gap-2 flex-shrink-0">
    <input type="checkbox" checked={attended} />
    <span>참석</span>
  </label>

  {/* 4. 삭제 버튼 (flex-shrink-0: 고정 크기) */}
  <button onClick={remove} className="flex-shrink-0">
    <Trash2 />
  </button>
</div>
```

### 시각적 계층 구조

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ┌──────────────────────┐  ┌─────┐┌────┐  ┌────┐  ┌────┐     │
│  │  이름 입력 (넓음)    │  │내부││팀장│  │참석│  │삭제│     │
│  └──────────────────────┘  └─────┘└────┘  └────┘  └────┘     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
     ↑                          ↑           ↑        ↑
   주요 입력               정보 표시      액션     관리
   (최대 공간)              (고정)       (고정)   (고정)
```

### CSS 클래스 구조

**Flexbox 설정**:
- `flex items-center gap-3`: 가로 정렬, 수직 중앙, 3단위 간격
- `flex-1`: 이름 입력 필드가 사용 가능한 공간 최대 차지
- `flex-shrink-0`: 배지, 체크박스, 버튼은 고정 크기 유지

**배지 스타일**:
```tsx
// 내부 직원 배지
<span className="text-xs font-medium text-blue-600 bg-blue-50 px-2 py-1 rounded">
  내부
</span>

// 외부 참석자 배지
<span className="text-xs font-medium text-gray-600 bg-gray-200 px-2 py-1 rounded">
  외부
</span>
```

## 📝 사용 시나리오

### 시나리오 1: 내부 직원 추가

```
1. "추가" 버튼 클릭
2. 이름 필드에 "최" 입력
   → 드롭다운: "최문호 (개발팀 팀장)" 표시
3. 선택
   → 결과:
   ┌────────────────────────────────────────────────┐
   │ [최문호] [내부][팀장] [참석☑] [삭제]          │
   └────────────────────────────────────────────────┘
```

**표시 내용**:
- 이름: "최문호" (자동완성으로 입력됨)
- 배지: "내부" (파란색)
- 직책: "팀장" (회색 텍스트)
- 참석: 체크됨 (기본값)

### 시나리오 2: 외부 참석자 추가

```
1. "추가" 버튼 클릭
2. 이름 필드에 "김고객" 입력
   → 드롭다운: '"김고객" 입력' 표시
3. 선택 또는 Enter
   → 결과:
   ┌────────────────────────────────────────────────┐
   │ [김고객] [외부] [참석☑] [삭제]                │
   └────────────────────────────────────────────────┘
```

**표시 내용**:
- 이름: "김고객" (수동 입력됨)
- 배지: "외부" (회색)
- 직책: 없음 (외부 참석자는 직책 불필요)
- 참석: 체크됨 (기본값)

### 시나리오 3: 혼합 참석자 목록

```
회의록 참석자:

┌──────────────────────────────────────────────────────┐
│ [최문호] [내부][팀장] [참석☑] [삭제]                │
├──────────────────────────────────────────────────────┤
│ [정회계] [내부][대리] [참석☑] [삭제]                │
├──────────────────────────────────────────────────────┤
│ [김고객] [외부] [참석☑] [삭제]                      │
├──────────────────────────────────────────────────────┤
│ [이협력] [외부] [참석□] [삭제]                      │
└──────────────────────────────────────────────────────┘
```

**특징**:
- 내부 직원: 직책 자동 표시
- 외부 참석자: 간결하게 "외부" 배지만 표시
- 불참자: 체크박스 해제 가능

## 🎯 UX 개선 효과

### 1. 입력 명확성 향상

**개선 전**:
```
입력 필드 2개:
[이름 입력]
[직책 입력 (선택사항)] ← 어디에 뭘 입력해야 하나?
```

**개선 후**:
```
입력 필드 1개:
[참석자 이름 검색 또는 입력...] ← 명확한 단일 입력점
```

### 2. 시각적 집중도

**이름 입력 필드**:
- `flex-1`로 최대 공간 차지
- 큰 입력 영역으로 시각적 주목
- Placeholder 텍스트가 명확히 보임

**정보 표시**:
- 배지 형태로 간결한 정보 전달
- 입력 필드와 구분되는 시각적 스타일
- 읽기 전용 정보로 혼동 방지

### 3. 공간 효율성

**2행 구조** → **1행 구조**:
- 세로 공간 50% 절약
- 더 많은 참석자를 한 화면에 표시
- 스크롤 감소

## 🔧 기술 세부사항

### 조건부 렌더링

```tsx
{participant.name && (
  <div className="flex items-center gap-2 flex-shrink-0">
    {participant.is_internal ? (
      // 내부 직원: 배지 + 직책
      <>
        <span className="badge-internal">내부</span>
        {participant.role && (
          <span className="text-sm">{participant.role}</span>
        )}
      </>
    ) : (
      // 외부 참석자: 배지만
      <span className="badge-external">외부</span>
    )}
  </div>
)}
```

**로직**:
1. `participant.name` 있을 때만 배지 표시
2. `is_internal === true`: 내부 배지 + 직책 표시
3. `is_internal === false`: 외부 배지만 표시
4. `role` 있을 때만 직책 텍스트 표시

### 외부 참석자 직책 처리

**개선 전**:
```typescript
// 외부 참석자 선택 시
{
  name: selectedName,
  role: '',              // 빈 문자열
  employee_id: undefined,
  is_internal: false
}

// 그 후 별도 입력 필드에서 role 입력
[직책 입력 (선택사항)]  ← 혼동 요소
```

**개선 후**:
```typescript
// 외부 참석자 선택 시
{
  name: selectedName,
  role: '',              // 빈 문자열 (사용 안 함)
  employee_id: undefined,
  is_internal: false
}

// 배지만 표시, 직책 입력 불필요
[외부] ← 간결
```
