# 회의록 작성 페이지 2열 레이아웃 디자인

## 🎯 디자인 목표

세계 최고 수준의 UX를 갖춘 2열 구성으로 공간 효율성을 극대화하고, 정보 계층과 사용자 흐름을 최적화합니다.

## 📐 레이아웃 구조

### **전체 레이아웃 원칙**
- **왼쪽 열 (60%)**: 핵심 회의 정보 (기본 정보, 참석자, 안건)
- **오른쪽 열 (40%)**: 부가 정보 및 액션 (회의 요약, 사업장별 이슈)
- **반응형**: 태블릿 이하에서 1열로 자동 전환
- **시각적 계층**: 카드 간 명확한 구분, 일관된 여백

### **정보 그룹핑 전략**

#### 왼쪽 열 (Primary Column)
```
┌─────────────────────────────────┐
│ 1. 기본 정보                    │
│    - 제목                       │
│    - 날짜/유형 (2열)            │
│    - 장소/유형 (2열)            │
├─────────────────────────────────┤
│ 2. 참석자                       │
│    - 동적 리스트 (컴팩트)       │
├─────────────────────────────────┤
│ 3. 안건                         │
│    - 번호 + 제목/설명           │
│    - 데드라인/담당자 (2열)      │
└─────────────────────────────────┘
```

#### 오른쪽 열 (Secondary Column)
```
┌─────────────────────────────────┐
│ 4. 회의 요약                    │
│    - 전체 높이 최적화           │
├─────────────────────────────────┤
│ 5. 사업장별 이슈                │
│    - 수평 스크롤 방지           │
│    - 세로 레이아웃 최적화       │
└─────────────────────────────────┘
```

## 🎨 UX 최적화 포인트

### 1. **시각적 균형**
- 왼쪽: 입력 필드 중심 (회의 설정)
- 오른쪽: 텍스트 영역 중심 (내용 작성)
- 자연스러운 좌→우 정보 흐름

### 2. **공간 효율성**
- **기본 정보**: 날짜/유형, 장소/유형을 2열 그리드로 압축
- **참석자**: 컴팩트한 수평 레이아웃, 삭제 버튼 최소화
- **안건**: 제목/설명 우선, 데드라인/담당자는 하단 2열
- **사업장별 이슈**: 세로 스택 레이아웃으로 가독성 확보

### 3. **인터랙션 최적화**
- **Sticky 헤더**: 각 섹션 헤더 고정 (스크롤 시 컨텍스트 유지)
- **스크롤 독립성**: 양쪽 열 독립 스크롤 가능
- **시각적 피드백**: 호버, 포커스 상태 명확화
- **버튼 배치**: 추가 버튼은 섹션 헤더 우측 고정

### 4. **반응형 전환점**
```
- Desktop (1280px+): 60/40 2열 레이아웃
- Laptop (1024px+): 55/45 2열 레이아웃
- Tablet (768px+): 1열 레이아웃 (순차적)
- Mobile (<768px): 1열 + 컴팩트 UI
```

## 🔧 구현 세부사항

### **그리드 시스템**
```css
.two-column-layout {
  display: grid;
  grid-template-columns: 3fr 2fr; /* 60/40 비율 */
  gap: 1.5rem; /* 24px */
  align-items: start;
}

@media (max-width: 1024px) {
  .two-column-layout {
    grid-template-columns: 1fr;
  }
}
```

### **카드 스타일**
- 배경: `bg-white`
- 테두리: `border border-gray-200`
- 그림자: `shadow-sm` (미묘한 깊이감)
- 패딩: `p-6` (일관성)
- 라운드: `rounded-lg`

### **섹션 헤더**
- 폰트: `text-lg font-semibold text-gray-900`
- 하단 마진: `mb-4`
- 액션 버튼: `flex items-center justify-between`

### **입력 필드 스타일**
- 포커스: `focus:ring-2 focus:ring-blue-500 focus:border-blue-500`
- 라운드: `rounded-lg`
- 패딩: `px-3 py-2` (컴팩트) / `px-4 py-3` (텍스트 영역)

## 📊 정보 밀도 비교

| 섹션 | 기존 (1열) | 신규 (2열) | 개선율 |
|------|-----------|-----------|--------|
| 기본 정보 | 8줄 | 5줄 | **37.5%** ↓ |
| 참석자 | 넓은 카드 | 컴팩트 | **40%** ↓ |
| 안건 | 세로 길이 | 최적화 | **30%** ↓ |
| 전체 스크롤 | 4-5 화면 | 2-3 화면 | **50%** ↓ |

## 🎯 사용자 흐름

### **작성 순서 (자연스러운 시선 흐름)**
1. **왼쪽 상단**: 제목 입력
2. **왼쪽**: 날짜, 유형, 장소 설정
3. **왼쪽**: 참석자 추가
4. **왼쪽**: 안건 작성 (데드라인/담당자 포함)
5. **오른쪽 상단**: 회의 요약 작성
6. **오른쪽 하단**: 사업장별 이슈 추가
7. **하단**: 저장 버튼

### **시각적 무게 배분**
- 왼쪽: 구조적 정보 (60% 폭, 입력 필드 밀집)
- 오른쪽: 서술적 정보 (40% 폭, 텍스트 영역)
- 균형잡힌 정보 밀도로 인지 부하 최소화

## 💡 디자인 철학

### **정보 아키텍처**
> "관련된 정보는 가깝게, 다른 정보는 분리하되 논리적 흐름을 유지한다"

- **왼쪽**: 회의 "설정" (누가, 언제, 무엇을)
- **오른쪽**: 회의 "결과" (어떻게, 왜, 액션)

### **시각적 계층**
> "중요도에 따라 크기와 위치를 결정한다"

- **최상단**: 회의록 제목 (가장 중요)
- **왼쪽 상단**: 날짜/유형 (핵심 메타데이터)
- **오른쪽**: 요약 및 이슈 (부가 정보)

### **인터랙션 원칙**
> "최소한의 동작으로 최대한의 효율을 낸다"

- **버튼 위치**: 섹션 우측 상단 고정 (예측 가능)
- **입력 흐름**: 위→아래, 좌→우 (자연스러운 순서)
- **피드백**: 즉각적인 시각적 반응 (호버, 포커스)
